# 🤖 MegaBot

**Telegram-бот для автоматизации ведения каналов**

Собирает материалы из RSS-лент → обрабатывает AI → публикует по расписанию

## Как это работает

```
RSS-лента  →  парсинг + очистка HTML  →  перевод (Google Translate)
     →  форматирование + эмодзи/хештеги  →  превью в боте
           →  публикация / очередь  →  APScheduler (каждую минуту)
```

1. Пользователь добавляет RSS-источники к своему каналу
2. Запускает сбор — бот парсит ленты, чистит HTML, переводит при необходимости
3. Показывает готовый пост в превью: можно отредактировать или сразу опубликовать
4. Пост уходит напрямую **или** добавляется в очередь — планировщик публикует по расписанию

## Возможности

### 📡 RSS и управление каналами
- Добавление канала по `@username`, ссылке или пересланному сообщению
- До **5 каналов** на аккаунт (настраивается через ключ лицензии)
- До **10 RSS-лент** на канал с индивидуальным включением/отключением
- Мультивыбор источников для разового сбора
- Встроенный набор тематических лент: IT, новости, крипта, игры, юмор и др.
- Пауза/возобновление публикаций для каждого канала отдельно

### ✍️ Обработка текста
- Очистка HTML, нормализация пробелов и переносов
- Перевод нерусских материалов через Google Translate **с кэшированием**
- Форматирование: заголовок **жирным**, ключевая мысль *курсивом*
- Автоматические эмодзи и хештеги по тематике канала
- Ссылка на источник `(Подробнее: URL)` — опционально
- Умное обрезание текста до 1000 символов по границе предложения

### 📅 Публикация и очередь
- Превью поста с возможностью редактирования перед отправкой
- Публикация с фото из RSS или текст — с автофолбэком
- Настраиваемый интервал между постами на каждый канал
- Дедупликация: один и тот же URL не попадёт в канал дважды
- Режим модерации: посты требуют ручного одобрения

### 📊 Статистика и уведомления
- Суммарные публикации, просмотры, реакции, репосты по каждому каналу
- Уведомления владельца об ошибках (нет прав, бот удалён из администраторов)
- Защита от спама уведомлений — не чаще раза в 10 минут

## Быстрый старт

### 1. Клонирование

```bash
git clone https://github.com/YOUR_USERNAME/megabot.git
cd megabot
```

### 2. Виртуальное окружение

```bash
# Linux / macOS
python3 -m venv .venv && source .venv/bin/activate

# Windows
python -m venv .venv && .venv\Scripts\activate
```

### 3. Зависимости

```bash
pip install -r requirements.txt
```

### 4. Конфигурация

```bash
cp .env.example .env
# Откройте .env и заполните BOT_TOKEN и ADMIN_IDS
```

### 5. Запуск

```bash
python main.py
```

> Бот работает в режиме long-polling. Для продакшена рекомендуется запускать под `systemd` или в `Docker`.

## Переменные окружения

| Переменная | Обязательная | Описание |
|---|:---:|---|
| `BOT_TOKEN` | ✅ | Токен от [@BotFather](https://t.me/BotFather) |
| `ADMIN_IDS` | ✅ | Telegram ID администраторов через запятую |
| `DATABASE_URL` | ✅ | Строка подключения к БД (по умолчанию `sqlite:///bot.db`) |
| `LICENSE_PEPPER` | — | Соль для хэширования ключей активации |
| `TZ` | — | Временная зона, например `Europe/Moscow` |
| `LOG_LEVEL` | — | Уровень логов: `DEBUG` / `INFO` / `WARNING` / `ERROR` |
| `OPENAI_API_KEY` | — | Ключ OpenAI (опционально, иначе используется G4F) |

> ⚠️ Файл `.env` добавлен в `.gitignore`. Никогда не коммитьте реальные токены.

## Команды бота

### Пользователю

| Команда / кнопка | Описание |
|---|---|
| `/start` | Главное меню |
| `/my_channels` | Список каналов |
| `/add_channel` | Добавить канал |
| `/enter_key` | Ввести ключ активации |
| `📰 RSS-источники` | Управление лентами канала |
| `✍️ Создать пост` | Собрать посты из RSS и выбрать для публикации |
| `📝 Очередь` | Посты, ожидающие публикации |
| `⏰ Расписание` | Настроить интервал между постами |
| `⚙️ Настройки` | Эмодзи, хештеги, ссылка на источник, модерация |
| `⏸️ Пауза / ▶️ Старт` | Включить/выключить автопостинг |

| Команда | Описание |
|---|---|

## Система лицензий

| Тариф | Каналов | Срок |
|---|:---:|---|
| Без ключа | 0 | — |
| basic | до 3 | задаётся при генерации |
| pro | до 5+ | задаётся при генерации |

- Ключи хранятся в БД в виде **SHA-256 хэша** (с pepper-солью)
- При истечении плана активные каналы автоматически деактивируются
- Администратор генерирует ключ командой `/genkey [дни] [каналы]`
